2 # Java Native Instrumentation Framework
3 
4 ## Introduction
5 
6 JNIF is the first native Java bytecode rewriting library.
7 JNIF is a C++ library for decoding, analyzing, editing,
8 and encoding Java bytecode.
9 The main benefit of JNIF is that it can be plugged into a JVMTI agent for
10 instrumenting all classes in a JVM transparently, i.e.,
11 without connecting to another JVM and without perturbing the observed JVM.
12 Besides, JNIF can be used in stand-alone tools as well.
13 
14 JNIF includes a data-flow analysis for stack map generation,
15 a complication necessary for any library that provides editing and encoding
16 support for modern JVMs with split-time verification.
17 It is written in C++11,
18 in an object-oriented style similar to Java-based class rewriting APIs.

26 ## Installation
27 
28 The JNIF library can be used as a static library which can be then linked
29 with your project.
30 
31 To compile JNIF it is enough to "make" the library:
32 
33 ```sh
34 make
35 ```
36 
37 All sources of JNIF are contained in the `src` folder.
38 
39 To use JNIF classes, the header file `jnif.hpp` must be included.
40 
41 ```c++
42 #include <jnif.hpp>
43 ```
44 
45 Make sure to add the `src` folder to the include search directories so
46 your compiler can found JNIF's headers, *i.e.*, with the `-I` compiler flag
47 and then link your project against the `libjnif.a`.

49 ## Getting started
50 
51 This section shows common use cases of the JNIF library,
52 such as writing instrumentation code and analyzing class files,
53 thus giving an overview of the library.
54 We present the examples in an incremental fashion,
55 adding complexity in each example.
56 
57 In order to be able to work with class files, first they must me parsed.
58 Given a memoery buffer with a class file and its length,
59 the following snippet shows how to parse it.
60 
61 ```c++
62 const char* data = ...;
63 int len = ...;
64 
65 jnif::ClassFile cf(data, len);
66 ```
67 
68 JNIF's `ClassFile` class provides fields and methods for analyzing and
69 editing a Java class.
70 It contains the definition of each method and field declared in the Java class.
71 
72 Once a class file is correctly parsed and loaded it can be manipulated using
73 the methods and fields in `ClassFile`.
74 For instance, in order to write back the parsed class file in a new buffer,
75 the write method is used in conjunction with the `computeSize` method as
76 shown below.
77 
78 ```c++
79 const char* data = ...;
80 int len = ...;
81 jnif::ClassFile cf(data, len);
82 int newlen = cf.computeSize();
83 u1* newdata = new u1[newlen];
84 cf.write(newdata, newlen);
85 
86 // Use newdata and newlen
87 
88 delete [] newdata;
89 ```
90 
91 Putting all together,
92 the following listing shows how to read and write a class file.
93 
94 ```c++
95 // Decode the binary data into a ClassFile object
96 const char* data = ...;
97 int len = ...;
98 jnif::ClassFile cf(data, len);
99 
100 // Analyze or edit the ClassFile
101 ...
102 
103 // Encode the ClassFile into binary
104 int newlen = cf.computeSize();
105 u1* newdata = new u1[newlen];
106 cf.write(newdata, newlen);
107 
108 // Use newdata and newlen
109 ...
110 
111 // Free the new binary
112 delete [] newdata;
113 ```
114 
115 The `ClassFile` class has a collection of fields and methods which can
116 be used to discover the members of the class file.
117 The snippet below shows how to traverse all methods in a class
118 to dump their names and descriptors.
119 Note that every `jnif` class overloads the `operator<<` in order
120 send it to an `std::ostream`.
121 
122 ```c++
123 const char* data = ...;
124 int len = ...;
125 jnif::ClassFile cf(data, len);
126 
127 for (jnif::Method* m : cf.methods) {
128  cout << "Method: ";
129  cout << cf.getUtf8(m->nameIndex);
130  cout << cf.getUtf8(m->descIndex);
131  cout << endl;
132 }
133 ```
134 
135 The following listing shows how to find all constructors
136 (named `<init>` at Java bytecode level) in a class
137 and how to inject instrumentation, in the form of a call to a static method
138 `static void alloc(Object o)` of an analysis class,
139 at the beginning of each constructor.
140 
141 ```c++
142 ConstIndex mid = cf.addMethodRef(classIndex, "alloc", "(Ljava/lang/Object;)V");
143 
144 for (Method* method : cf.methods) {
145  if (method->isInit()) {
146  InstList& instList = method->instList();
147 
148  Inst* p = *instList.begin();
149  instList.addZero(OPCODE_aload_0, p);
150  instList.addInvoke(OPCODE_invokestatic, mid, p);
151  }
152 }
153 ```
154 
155 Another common use case is to instrument every method entry and exit.
156 In order to do so, it is possible to add the instrumentation code at the
157 beginning of the instruction list to detect the method entry.
158 To detect method exit,
159 it is necessary to look for instructions that terminate the current method
160 execution, i.e.,
161 xRETURN family and ATHROW as showed in the following snippet.
162 
163 ```c++
164 ConstIndex sid = cf.addMethodRef(proxyClass, "enterMethod",
165  "(Ljava/lang/String;Ljava/lang/String;)V");
166 ConstIndex eid = cf.addMethodRef(proxyClass, "exitMethod",
167  "(Ljava/lang/String;Ljava/lang/String;)V");
168 ConstIndex classNameIdx = cf.addStringFromClass(cf.thisClassIndex);
169 
170 ...
171 
172 InstList& instList = method->instList();
173 
174 ConstIndex methodIndex = cf.addString(m->nameIndex);
175 
176 Inst* p = *instList.begin();
177 
178 instList.addLdc(OPCODE_ldc_w, classNameIdx, p);
179 instList.addLdc(OPCODE_ldc_w, methodIndex, p);
180 instList.addInvoke(OPCODE_invokestatic, sid, p);
181 
182 for (Inst* inst : instList) {
183  if (inst->isExit()) {
184  instList.addLdc(OPCODE_ldc_w, classNameIdx, inst);
185  instList.addLdc(OPCODE_ldc_w, methodIndex, inst);
186  instList.addInvoke(OPCODE_invokestatic, eid, inst);
187  }
188 }
189 ```
190 
191 Besides providing access to all members of a class,
192 `ClassFile` also provides access to the constant pool
193 via methods like `getUtf8()` and `addMethodRef()`.

195 ## Test and evaluation
196 
197 To run these evaluations, a Makefile script is provided in the git repository.
198 These tasks take care of the compilation of the JNIF library and also
199 all java files needed.
200 The repository is self-contained,
201 no need to download dacapo benchmarks separately.
202 
203 ```sh
204 > make testapp
205 ```
206 
207 To run a particular dacapo benchmark with default settings:
208 
209 ```sh
210 > make dacapo BENCH=avrora
211 ```
212 
213 To run a full evaluation with all dacapo benchmarks in all configuration a
214 task -eval- is provided.
215 You can set how many times run each configuration with the variable times, like
216 
217 ```sh
218 > make eval times=5
219 ```
220 
221 Finally, there is a task to create plots for the evaluation.
222 This task needs R with the package ggplot2.
223 
224 ```sh
225 > make plots
226 ```
