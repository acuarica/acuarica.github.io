4 # Introduction
5 
6 JNIF is the first native Java bytecode rewriting library.
7 JNIF is a C++ library for decoding, analyzing, editing,
8 and encoding Java bytecode.
9 The main benefit of JNIF is that it can be plugged into a JVMTI agent for
10 instrumenting all classes in a JVM transparently, i.e.,
11 without connecting to another JVM and without perturbing the observed JVM.
12 Besides, JNIF can be used in stand-alone tools as well.
13 
14 JNIF includes a data-flow analysis for stack map generation,
15 a complication necessary for any library that provides editing and encoding
16 support for modern JVMs with split-time verification.
17 It is written in C++11,
18 in an object-oriented style similar to Java-based class rewriting APIs.

26 # Installation
27 
28 The JNIF library can be used as a static library which can be then linked
29 with your project.
30 
31 To compile JNIF it is enough to "make" the library
32 
33  make
34 
35 All sources of JNIF are contained in the "src" folder.
36 
37 To use JNIF classes, the header file **jnif.hpp** must be included.
38 
39  #include <jnif.hpp>
40 
41 Make sure to add the **src** folder to the include search directories so
42 your compiler can found JNIF's headers, i.e., with the **-I** compiler flag
43 and then link your project against the **libjnif.a**.

45 # Getting started
46 
47 This section shows common use cases of the JNIF library,
48 such as writing instrumentation code and analyzing class files,
49 thus giving an overview of the library.
50 We present the examples in an incremental fashion,
51 adding complexity in each example.
52 
53 In order to be able to work with class files, first they must me parsed.
54 Given a memoery buffer with a class file and its length,
55 the following snippet shows how to parse it.
56 
57  const char* data = ...;
58  int len = ...;
59 
60  jnif::ClassFile cf(data, len);
61 
62 JNIF's **ClassFile** class provides fields and methods for analyzing and
63 editing a Java class.
64 It contains the definition of each method and field declared in the Java class.
65 
66 Once a class file is correctly parsed and loaded it can be manipulated using
67 the methods and fields in **ClassFile**.
68 For instance, in order to write back the parsed class file in a new buffer,
69 the write method is used in conjunction with the **computeSize** method as
70 shown below.
71 
72  const char* data = ...;
73  int len = ...;
74  jnif::ClassFile cf(data, len);
75  int newlen = cf.computeSize();
76  u1* newdata = new u1[newlen];
77  cf.write(newdata, newlen);
78 
79  // Use newdata and newlen
80 
81  delete [] newdata;

107 The **ClassFile** class has a collection of fields and methods which can
108 be used to discover the members of the class file.
109 The snippet below shows how to traverse all methods in a class
110 to dump their names and descriptors.
111 Note that every **jnif** class overloads the **operator&lt;&lt;** in order
112 send it to an **std::ostream**.
113 
114  const char* data = ...;
115  int len = ...;
116  jnif::ClassFile cf(data, len);
117 
118  for (jnif::Method* m : cf.methods) {
119  cout << "Method: ";
120  cout << cf.getUtf8(m->nameIndex);
121  cout << cf.getUtf8(m->descIndex);
122  cout << endl;
123  }
124 
125 The following listing shows how to find all constructors
126 (named **&lt;init&gt;** at Java bytecode level) in a class
127 and how to inject instrumentation, in the form of a call to a static method
128 **static void alloc(Object o)** of an analysis class,
129 at the beginning of each constructor.
130 
131  ConstIndex mid = cf.addMethodRef(classIndex, "alloc", "(Ljava/lang/Object;)V");
132 
133  for (Method* method : cf.methods) {
134  if (method->isInit()) {
135  InstList& instList = method->instList();
136 
137  Inst* p = *instList.begin();
138  instList.addZero(OPCODE_aload_0, p);
139  instList.addInvoke(OPCODE_invokestatic, mid, p);
140  }
141  }
142 
143 Another common use case is to instrument every method entry and exit.
144 In order to do so, it is possible to add the instrumentation code at the
145 beginning of the instruction list to detect the method entry.
146 To detect method exit,
147 it is necessary to look for instructions that terminate the current method
148 execution, i.e.,
149 xRETURN family and ATHROW as showed in the following snippet.
150 
151  ConstIndex sid = cf.addMethodRef(proxyClass, "enterMethod",
152  "(Ljava/lang/String;Ljava/lang/String;)V");
153  ConstIndex eid = cf.addMethodRef(proxyClass, "exitMethod",
154  "(Ljava/lang/String;Ljava/lang/String;)V");
155  ConstIndex classNameIdx = cf.addStringFromClass(cf.thisClassIndex);
156 
157  ...
158 
159  InstList& instList = method->instList();
160 
161  ConstIndex methodIndex = cf.addString(m->nameIndex);
162 
163  Inst* p = *instList.begin();
164 
165  instList.addLdc(OPCODE_ldc_w, classNameIdx, p);
166  instList.addLdc(OPCODE_ldc_w, methodIndex, p);
167  instList.addInvoke(OPCODE_invokestatic, sid, p);
168 
169  for (Inst* inst : instList) {
170  if (inst->isExit()) {
171  instList.addLdc(OPCODE_ldc_w, classNameIdx, inst);
172  instList.addLdc(OPCODE_ldc_w, methodIndex, inst);
173  instList.addInvoke(OPCODE_invokestatic, eid, inst);
174  }
175  }

177 Besides providing access to all members of a class,
178 **ClassFile** also provides access to the constant pool
179 via methods like **getUtf8()** and **addMethodRef()**.

181 # Test and evaluation
182 
183 To run these evaluations, a Makefile script is provided in the git repository.
184 These tasks take care of the compilation of the JNIF library and also
185 all java files needed.
186 The repository is self-contained,
187 no need to download dacapo benchmarks separately.
188 
189  > make testapp
190 
191 To run a particular dacapo benchmark with default settings:
192 
193  > make dacapo BENCH=avrora
194 
195 To run a full evaluation with all dacapo benchmarks in all configuration a
196 task -eval- is provided.
197 You can set how many times run each configuration with the variable times, like
198 
199  > make eval times=5
200 
201 Finally, there is a task to create plots for the evaluation.
202 This task needs R with the package ggplot2.
203 
204  > make plots
