Language Ideas

These are some requirements that a language should have.

  • No Abstraction Penalty. As In Rust/C++
  • Embed Optimizations within the language itself.
  • Functional objects as in Lisp: S-Expressions, S-Functions.
  • But it should be carefull about compatibility:
    • Idioms => Compatible
    • vs.
    • Dialects => Non-Compatible
  • Types as Sets? Propositions. Better to Use Syntatic Types? Constructive/Intuitionistic Logic
  • Co-routines, Generators (yield KW as in Python, C#), continuations. C co-routines “switch” statement. “The Art of Programming”, D. Knuth.
  • Namespaced keyword Different implementations for
    • Classes
    • Closures
    • functions-methods
    • yield-coroutines
  • Typestates for Objects, Typestate, Typechecking, Typestate-oriented type qualifiers.
  • Data abstraction, subtyping?
  • Concurrency, Data races
  • Instrumentation, Reflection, Aspect-Oriented Programming
  • Operational Dependent Types. Sorted List

For example, within list l, be able to constraint l such as:

for i = 0 .. n-1:
  l[i] <= l[i+1]
